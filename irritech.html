IRRITECH

Système d'arrosage automatique intelligent

PROTOTYPAGE

INTRODUCTION

IRRITECH est un système d'arrosage automatique qui va arroser les plantes de manière autonome. Si la plante a besoin d'eau, le système le saura et procédera à l'arrosage. C'est un projet de grande envergure destiné à être installé dans les villages et les champs.

Une version plus petite a déjà été conçue par l'équipe AFRICA.TECH21, mais cette nouvelle version doit être adaptée aux grandes surfaces agricoles.

ÉTUDE DU TERRAIN / CLIMAT

Pour que le système fonctionne dans de bonnes conditions, plusieurs points doivent être respectés :

  • Protection contre les enfants : L'appareil ne doit pas être à la portée des enfants ou des personnes non qualifiées.
  • Zone non humide : Les composants électroniques ne doivent pas être dans une zone humide.
  • Terrain stable : Le terrain ne doit pas être très accidenté, sinon risque de perturber le système.
  • Protection contre les éléments : Les composants internes doivent être protégés contre la poussière, la saleté, le sable, l'humidité, les températures extrêmes.
  • Fertilité du sol : Il faut tester la fertilité du sol.

NB : Il n'est pas obligatoire de faire des études sur le rendement du sol, car cette action est déjà faite par les agriculteurs. Nous installons le système en tenant compte des conditions obligatoires pour le bon fonctionnement d'IRRITECH.

ÉTUDE DES PLANTES

Cette étape est délicate et demande une attention profonde. Voici ce qui doit être fait :

  • Étude systématique de chaque variété : carottes, pomme de terre, oignon, salade, tomates, etc.
  • Tolérance en eau : voir la tolérance en eau de chaque plante, ainsi que le seuil de température (climat).
  • Durée d'évolution : la durée d'évolution de chaque espèce.
  • Qualité des semences : la qualité des semences.

Après avoir recueilli toutes les informations nécessaires, elles seront converties en une suite d'instructions (algorithmes) et interprétées par des capteurs et actionneurs.

Attention : Les conditions climatiques doivent être favorables pour la bonne germination des plantes. Dans le cas contraire, il faudra recourir à des méthodes comme les "serres" ou adapter le système, ce qui n'est pas évident.

COMPOSANTS ÉLECTRONIQUES

Objectif : Une carte mère très performante contrôle tous les composants du projet :

  • Capteurs : sondes, lecteurs, etc.
  • Actionneurs : pompes submersibles, électrovannes, valves.
  • Relais : pour fournir une alimentation indépendante aux actionneurs, évitant d'encombrer le microcontrôleur. Le choix des relais dépend de la tension et du courant des actionneurs.

Choix de la carte mère

Deux options sont envisagées : ESP32 ou Raspberry Pi mini pour des performances élevées.

Capteurs d'humidité du sol (Soil Moisture V1.2)

Le capteur d'humidité du sol Soil Moisture V1.2, fabriqué par ESP32, est un appareil fiable conçu pour mesurer la teneur en eau volumétrique du sol. Il fournit des données précises pour la gestion de l'irrigation.

  • Protection : Équipé d'un revêtement de résine isolant le circuit de détection. Les composants électroniques doivent être isolés à l'aide de gaine électrique ou de résine époxy.
  • Complémentarité : Le DHT22 peut être utilisé pour détecter la température et l'humidité en parfaite complémentarité avec le V1.2.
  • Installation : Les capteurs sont répartis en blocs de 4 capteurs sur un rectangle de longueur 1,70 m et de largeur 1,20 m. Le nombre de blocs dépend de la surface du jardin.

Carte ESP32

La carte ESP32 dispose de 15 à 18 entrées analogiques (ADC), réparties sur deux convertisseurs (ADC1 et ADC2), offrant une résolution de 12 bits (0-4095). Elle permet de contrôler jusqu'à 4 blocs IRRITECH.

Problème des ADC : Les ADC sont réputées pour avoir des fluctuations ou du bruit, ce qui peut causer des lectures fausses. Pour corriger ce défaut, on peut ajouter un condensateur à la sortie et utiliser du suréchantillonnage.

Condensateur : Un condensateur céramique d'une capacité de 100 nF (0,1 µF) est recommandé pour lisser les pics.

Pompe électrique

La pompe électrique doit fournir une pression ou un débit assez élevé pour assurer l'arrosage. La puissance recommandée est de 12 VDC minimum à 24 VDC maximum en courant continu.

PARTIE SOFTWARE

La partie software donne vie au projet. Sans un programme bien défini et prêt à être téléversé, le hardware ne sert à rien.

Langage de programmation : Python

La communication avec la carte ESP32 se fait via le langage Python (MicroPython), populaire et doté d'une forte communauté.

IDE : Thonny

Thonny est un environnement de développement intégré (IDE) libre et open-source, spécifiquement conçu pour simplifier l'apprentissage et l'usage de Python et MicroPython. Il est devenu la référence pour le prototypage rapide sur microcontrôleurs comme l'ESP32 ou le Raspberry Pi Pico.

Contrairement à l'IDE Arduino, Thonny permet de contrôler beaucoup de composants électroniques sans avoir à installer de bibliothèques.

MicroPython

MicroPython est une implémentation optimisée du langage Python 3, conçue pour s'exécuter sur des microcontrôleurs comme l'ESP32. Il permet de piloter du matériel avec une syntaxe simple, sans gérer la complexité du C ou C++.

Prompt du programme

Le programme doit gérer 4 capteurs V1.2 connectés respectivement aux GPIO 32, 33, 34, 35 de l'ESP32.

Configuration des pins (extrait de code)

from machine import ADC, Pin
import time

# Configuration des 4 capteurs sur ADC1 (GPIO 32, 33, 34, 35)
c1 = ADC(Pin(32))
c2 = ADC(Pin(33))
c3 = ADC(Pin(34))
c4 = ADC(Pin(35))

Explications :

  • from machine import ADC, Pin : Importe le module qui permet de contrôler le hardware (processeur).
  • ADC : Convertisseur Analogique-Digital. Transforme la tension du capteur (0 V à 3.3V) en un nombre (0 à 4095).
  • Pin : Permet de désigner physiquement les broches de l'ESP32.
  • c1, c2, c3, c4 : Objets associés aux capteurs sur les broches GPIO 32, 33, 34, 35.
  • import time : Sert à régler le temps (comme delay en C++).

Calcul de la moyenne pondérée

Les 4 capteurs donnent des informations de leur côté. Pour éviter qu'une seule zone plus sèche déclenche l'arrosage inutilement, on fait la moyenne des 4 lectures pour obtenir une image globale de la parcelle.

def obtenir_moyenne():
    # On lit chaque capteur
    v1 = c1.read()
    v2 = c2.read()
    v3 = c3.read()
    v4 = c4.read()
    moyenne = (v1 + v2 + v3 + v4) / 4
    return moyenne

Condition d'arrosage

while True:
    humidite_globale = obtenir_moyenne()
    print("Niveau moyen d'humidité :", humidite_globale)

    if humidite_globale < 1500: # Seuil à calibrer
        print("Alerte : Sol sec ! Activation de l'arrosage.")
        # Code pour activer la pompe
    time.sleep(60) # Attendre 1 minute entre chaque lecture
